# 安全政策 / Security Policy

## 報告安全問題 / Reporting Security Issues

如果你發現本項目中嘅安全問題，請透過以下方式私下報告：

If you discover a security issue in this project, please report it privately via:

### 首選方法 / Preferred Method

**GitHub 私人漏洞報告 / GitHub Private Vulnerability Reporting**

透過 GitHub 嘅私人漏洞報告功能報告：[報告漏洞](https://github.com/hklittlefinger/wang-fuk-court-fire-investigation/security/advisories/new)

Report via GitHub's private vulnerability reporting: [Report a vulnerability](https://github.com/hklittlefinger/wang-fuk-court-fire-investigation/security/advisories/new)

### 替代方法 / Alternative Method

**電郵 / Email**: hklittlefinger@gmail.com

主題請註明「Security」或「安全」。

Please include "Security" or "安全" in the subject line.

## 範圍 / Scope

本安全政策涵蓋：

This security policy covers:

### 代碼同基礎設施 / Code and Infrastructure
- AWS 啟動同監控腳本 (`simulation/scripts/`)
- 任何處理敏感數據嘅代碼
- 基礎設施配置

- AWS launch and monitoring scripts (`simulation/scripts/`)
- Any code that handles sensitive data
- Infrastructure configurations

### 證據檔案 / Evidence Files
- 未遮蔽嘅個人身份資訊（面部、姓名、地址）
- 圖片或文件中嵌入嘅惡意軟件
- 暴露貢獻者身份嘅元數據

- Unredacted personally identifiable information (faces, names, addresses)
- Malware embedded in images or documents
- Metadata exposing contributor identities

## 回應時間 / Response Time

我哋會盡快回應安全報告，通常喺 72 小時內。

We will respond to security reports as soon as possible, typically within 72 hours.
